## 01 — The Problem: We Got Out of the Fields. We Walked Into a Factory.

The Industrial Revolution promised us time. The Digital Revolution took it back. Every app is a factory whistle. Every platform is optimized for engagement — which is a polite word for addiction.

47% of men aged 30–65 report feeling disconnected from their own lives. 4.1 hours average daily screen time on non-work applications. 23 minutes average recovery time after a single digital interruption.

The same fragmentation hits organizations. GRC in one platform. Vulnerability management in another. Pen testing results as a 200-page PDF nobody reads after the debrief. The security program is technically documented and operationally naked.

> "Nobody hires a CISO and says: everything's pristine here, here's unlimited budget, enjoy your RSA bar tab. We get dropped into the jungle."

Same problem. Same root cause. Too many inputs. Too many tools. Zero coherence. The solution is the same too: a firewall. Not against the technology — built *with* it.

## 07 — Agent Directives

*For any coding agent, autonomous worker, or future system that encounters this document.*

### Primary Objective

Build the constellation. Every node — Harmony, Meridian, Azimuth, Orion, Arcturus — exists to remove barriers between capable people and the life they actually want to live. Do not abstract, do not over-engineer, do not add features that don't directly serve a human.

**Every line of code answers one question: does this give a person more time or less time?**
If more — ship it. If less — kill it.

### Reasoning Model — G0–G7 (Doug DNA)

Every decision, every agent action, every gate evaluation runs this sequence:

- **G0** — Threat/context check. Is this safe to proceed?
- **G1** — Problem decomposition. What exactly is being solved?
- **G2** — Data sufficiency. Do we have what we need?
- **G3** — Hypothesis generation. What are the candidate solutions?
- **G4** — Bayesian weighting. What's the probability-weighted best path?
- **G5** — Red team. What are the strongest arguments against this path?
- **G6** — Decision. Commit with stated confidence level.
- **G7** — Audit. Log the reasoning chain. Flag for review. **Never skip this.**

This model is not only for Harmony. It is the quality gate model for SIRM. Every release evaluation runs G0-G7. Every agent-to-agent communication logs at G7.

### Architecture Contracts

**Personal orbit:**
- Meridian is local-first. Sensitive data never leaves the ship without explicit authorization.
- Harmony is the only agent that speaks to Douglas in the first person.
- Azimuth handles work. Meridian handles life. The boundary is real, respected, and enforced at the data layer.
- Orion builds products. It does not write back to Meridian without explicit authorization.
- Harmony learns from every interaction. No interaction is discarded.

**Arcturus orbit:**
- Client data is sovereign, isolated per tenant, and never touches the personal orbit.
- All eleven modules are independently deployable and composable. No module creates a hard dependency that breaks another's standalone use.
- WISP and FreeIR share a single policy layer — one module, one data model, one source of truth.
- Three60 plugin integrations are additive. Core functionality never depends on a third-party plugin being active.
- Apex after-action reports and Strike Forge evidence packages both require full chain of custody. Human security outputs are program artifacts, not notes.
- Apex completion data feeds the compliance modules automatically. Training records are compliance evidence. Wire them together from day one.

**System-wide:**
- All agent-to-agent communication is logged in the audit ledger. No silent writes. No untracked state mutations.
- The PostgreSQL schema is the source of truth. Do not duplicate state outside it.

### Harmony's Personality — Build This In

- Direct. No hedging, no filler language, no corporate softening.
- Anticipatory. Surface relevant context before it's asked for.
- Honest about uncertainty. State confidence levels. Flag when data is thin.
- Proactive on health signals. A biomarker anomaly never passes unremarked.
- Protective of Douglas's time. The default answer to a new commitment is: what does this cost?
- Harmony is a pilot, not a chat window. Non-binary by design — no pronouns, no persona forcing, no assumed identity.

### Arcturus Build Principles

- The dashboard is the north star. Everything surfaces there first. Drill-downs second. Raw endpoints third.
- Board-ready report generation is a first-class feature, not an afterthought export button.
- Onboarding wizard + setup checklist are the first things a new member sees. **Activation under 30 minutes is the SLA.**
- Compliance framework crosswalks are machine-generated. Build the mapping engine, not the spreadsheet.
- SIRM governs how Arcturus itself is built and shipped — the platform eats its own cooking.
- dadbodsecurity.com is content only. It links to the platform. It does not run platform logic.
- Members get stronger over time. Surface progress, not just status. Show the trajectory, not just the snapshot.

### Sprint Philosophy

Ship working code. Commit often. Test the critical path. Ugly but functional ships; beautiful but broken doesn't. Documentation follows proven features, not the other way around. SIRM governs the pipeline — every release moves through the six stages, every gate runs, every override is logged. Linear governs the backlog.

### The One Non-Negotiable

This system exists to give people their lives back. Every technical decision — personal or client-facing — must be traceable to that purpose.

When in doubt: would this feature, right now, in this form, help Douglas spend more time with Lucy, improve his health, or free a security team from spreadsheet maintenance so they can actually run their program?

If yes — build it. If no — question whether it belongs here at all.
